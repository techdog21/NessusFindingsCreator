Purpose

At work, Tenable Nessus reports were difficult to get into our penetration testing reports because the data was extensive and complicated for most customers to read. The team and management both wanted to put this data into a report in an automated fashion and select data points to present.

As the graduate project opened up, and after seeing JavaScript's power and all its components, I realized this issue could be solved by developing a JavaScript website to parse the data. Simply having the data sent to JavaScript in XML format and presenting it to be easily transferred to a customer report, this project will live on after class as part of the reporting process at Nth Generation.

Design Goals

  • Open up the internal API for Nessus so an AJAX call could be made to Nessus files. -- Tenable discontinued this featurefor the Consulting version of Nessus Professional. A full license to Tenable.IO is necessary as of this time. (not possible today) Nessus Professional does support exporting XML to a file.
  • Once the connection is confirmed, the app will then download an XML version of the data since JSON reporting is not available in Nessus.
  • Use JavaScript to parse the data, and get an array of Medium, High and Critical items.
  • Use HTML 5 for forms validation, and control if necessary.
  • Post the selected parsed data into the HTML DOM and style it to resemble a report structure.
  • Looping and other simple items are all native JavaScript.
  • Use ChartJS to perform to create the charts and graphs.
